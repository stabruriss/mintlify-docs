The Vibrant Wellness Portal API is a REST API. All endpoints accept and return JSON, support bearer authentication, and emit signed webhooks for asynchronous events.

## Base URLs

| Environment | URL |
| --- | --- |
| Production | `https://api.vibrant-wellness.example.com` |
| Sandbox | `https://api.sandbox.vibrant-wellness.example.com` |

All paths shown in this reference are versioned with `/v1/`.

## Authentication

Every request requires an `Authorization` header:

```http
Authorization: Bearer sk_live_01HZ7M3K2N8R9Q...
```

Generate keys in the portal. See [Authentication](/authentication) for scope and rotation guidance.

## Conventions

<CardGroup cols={2}>
  <Card title="Identifiers" icon="fingerprint">
    Prefixed ULIDs: `ord_...`, `rpt_...`, `pat_...`, `prv_...`, `tst_...`, `evt_...`. URL-safe and case-sensitive.
  </Card>
  <Card title="Timestamps" icon="clock">
    All timestamps are RFC 3339 / ISO 8601 in UTC, e.g. `2026-05-04T14:22:11Z`.
  </Card>
  <Card title="Pagination" icon="list">
    All list endpoints use cursor-based pagination. See [Pagination](/platform/pagination).
  </Card>
  <Card title="Errors" icon="triangle-exclamation">
    Errors share a single envelope. See [Errors](/platform/errors).
  </Card>
</CardGroup>
