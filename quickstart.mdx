This guide walks you from a fresh Vibrant Wellness account to a placed sandbox order. You'll need a portal account with the **Developer** role.

<Steps>
  <Step title="Generate an API key">
    Open the portal and go to **Settings → API keys**. Click **Create key**, give it a name (e.g. `local-dev`), and choose the **Sandbox** environment.

    <Frame>
      <img src="/images/checks-passed.png" alt="API keys page in the Vibrant Wellness portal" />
    </Frame>

    Copy the key immediately — it's shown once. Treat it like a password.

    <Tip>
      Need a walk-through with mock UI? See the [API key demo](/api-keys).
    </Tip>
  </Step>

  <Step title="Export your key and the sandbox base URL">
    ```bash
    export VW_API_KEY="sk_sandbox_..."
    export VW_API_BASE="https://api.sandbox.vibrant-wellness.example.com"
    ```
  </Step>

  <Step title="Verify auth with a catalog call">
    The test catalog is the cheapest endpoint to confirm you're authenticated.

    <CodeGroup>
      ```bash cURL
      curl "$VW_API_BASE/v1/tests?limit=3" \
        -H "Authorization: Bearer $VW_API_KEY"
      ```

      ```ts TypeScript
      const res = await fetch(`${process.env.VW_API_BASE}/v1/tests?limit=3`, {
        headers: { Authorization: `Bearer ${process.env.VW_API_KEY}` },
      });
      const { data } = await res.json();
      console.log(data);
      ```

      ```python Python
      import os, requests

      r = requests.get(
          f"{os.environ['VW_API_BASE']}/v1/tests",
          params={"limit": 3},
          headers={"Authorization": f"Bearer {os.environ['VW_API_KEY']}"},
      )
      r.raise_for_status()
      print(r.json()["data"])
      ```
    </CodeGroup>

    A 200 response with a `data` array means you're in.
  </Step>

  <Step title="Place a sandbox order">
    The sandbox accepts the same payloads as production but never ships a real kit or charges a real card.

    ```bash
    curl -X POST "$VW_API_BASE/v1/orders" \
      -H "Authorization: Bearer $VW_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "patient": {
          "external_id": "demo_patient_1",
          "first_name": "Avery",
          "last_name": "Nguyen",
          "date_of_birth": "1989-03-14",
          "sex": "female",
          "shipping_address": {
            "line1": "1200 Market St",
            "city": "San Francisco",
            "state": "CA",
            "postal_code": "94103",
            "country": "US"
          }
        },
        "provider_id": "prv_demo",
        "tests": [{ "test_code": "GUT-ZOOMER-3.0" }],
        "sample_collection": { "method": "home_kit" }
      }'
    ```

    You'll get back an order resource with an `id` like `ord_01HZ7M3K2N8R9Q` and `status: "created"`.
  </Step>

  <Step title="Watch it move">
    Sandbox orders auto-advance through every status on a fast clock so you can build tracking UI without waiting on a real lab. Poll the tracking endpoint or subscribe to webhooks:

    ```bash
    curl "$VW_API_BASE/v1/orders/ord_01HZ7M3K2N8R9Q/tracking" \
      -H "Authorization: Bearer $VW_API_KEY"
    ```

    See [Track an order](/guides/track-order) for the full timeline.
  </Step>
</Steps>
