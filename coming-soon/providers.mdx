Today, providers (`provider_id` values) are provisioned by Vibrant's onboarding team and shared with you out of band. The upcoming Providers API will let your account admins create, update, and revoke providers programmatically — useful for clinics with high turnover or for SaaS partners shipping multi-tenant integrations.

## Planned endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/v1/providers` | Create a provider. |
| `GET` | `/v1/providers` | List providers. |
| `GET` | `/v1/providers/{provider_id}` | Retrieve. |
| `PATCH` | `/v1/providers/{provider_id}` | Update mutable fields. |
| `POST` | `/v1/providers/{provider_id}/deactivate` | Deactivate. |

## Validation expectations

- `npi` must be a valid 10-digit National Provider Identifier (Luhn-checked).
- Each `licenses[].state` must match an active license in the relevant state board's registry.
- `email` must match the domain on file for the practice (configurable).

Failed validations will return `validation_error` with `param` pointing to the offending field.
